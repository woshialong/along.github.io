敏捷开发各个阶段业务测试人员的知识体系

发表于 2020-03-04 | 更新于 2020-03-05 | 分类于 学习笔记

最近在协助一些同学制定一些个人成长计划，感觉测试人员综合能力涉及的知识点很多，于是想整理一份相关文档，提供自己和他人规划学习方向。我打算以需求生命周期作为维度，从需求阶段、开发阶段、测试阶段和发布上线四个阶段来梳理测试人员工作职责和一些技术栈的运用，当然肯定也会有些不全面，后续有机会定期再做增补。。。

1.先放一个脑图

2.需求阶段

测试左移，测试人员要有对产品需求和技术设计有分析和提出观点的能力和意识

2.1 需求评审

对自身业务线产品有清晰的认识，了解且有竞品分析意识，基础需求交互设计的能力，需求评审时我们通常对需求提出意见会考虑一下三个维度：

  • 对核心业务KPI的作用（产品价值）
  • 用户体验（用户角度）
  • 技术可行性（结合前后端技术实现）

2.2 设计评审

测试人员要有阅读设计文档的能力，理清自身业务详细架构图，表结构，接口全链路。设计阶段，可以对一些例如redis缓存策略，表结构设计，接口前端接入是否方便，功能聚合还是解耦提出自己的观点。

3.开发阶段

开发阶段时，测试人员主要是在设计用例阶段。再次期间测试可以做很多事情：

  • 提供冒烟用例供开发自测，协助开发梳理主链路case，同时反推提测质量
  • code review代码能力，在提测初期，通过快速的代码review。评估case覆盖面和是否有性能风险。比如底层接口改动对上游哪些业务有影响，是否有表结构变动，对端上是否会有新老版本兼容问题等。对于本次改动是否涉及大流量、大数据量或者复杂的代码逻辑造成的性能风险。
  • 静态扫描 单元测试覆盖率：这里说的粗一些，首先这一块工作会和开发团队配合沟通比较多，选择jacoco findbugs框架或者sonarqube平台，接入提测阶段的持续集成工作流中 ，把增量代码质量评估加入到整体流程

4.测试阶段

在开发代码提测通过后，进入了常规用例测试执行阶段，这期间也有不同维度的技术手段提高质效。

4.1 自动化建设提效

我们把自动化建设分几个方向进行讲解

  • 工具选型：根据团队配置选择合适的框架，需要评估效率、团队能力、二次开发量
  • 用例脚本设计思路：根据相关业务实际情况，按接口维度设计脚本；按场景维度设计脚本；UI自动化按页面&组件维度设计脚本；抽象suite套件、封装工具类供重复调用避免重复造轮子
  • 执行场景：集成到测试提测冒烟、预发和生产发布后回归，手工抽取脚本跑批，定期线上健康检查定时任务

4.2 服务端专项

  • 接口自动化-常规框架、用例设计思路（按接口、按场景、上下游mock的运用）、封装通用函数工具类
  • 静态扫描 单元测试覆盖率 服务端代码语法
  • java技术栈相关知识和服务端整体链路清晰，springmvc+dubbo微服务框架，数据库、消息队列等
  • 对日志服务监控和实例监控的运用，建立监控告警问题反馈和跟进解决的流程，形成闭环

4.3 前端专项

  • 兼容性测试、稳定性测试：常用框架、云测平台，平台化和接入持续集成
  • UI自动化：常规框架（selenium、appium、webdriver、无头浏览器）、设计思路（组件化下的前端项目，按页面和组件的覆盖）
  • 前端异常上报和前端性能：页面加载生命周期、前端上报异常和跟进解决的流程，形成闭环

4.4 性能专项

  • 压测工具：常用工具jmeter、nginder、locust
  • 接口和常规场景压测：场景设计能力；指标分析和多维度监控能力
  • 性能问题排查和优化：服务端架构和分析能力：jvm、数据库性能、redis、消息队列、异步处理、nginx、网关、负载均衡等
  • 全链路压测：流量录制回放能力，服务高可用要求（稳定性相关测试），压测脏数据处理，健壮的监控能力

4.5 稳定性保障

混沌测试、故障演练。提高稳定性、服务高可用和容灾，具体操作步骤：

  • 模拟服务crash、网络异常、第三方服务不可用
  • 分析服务健壮性，减少影响面（解耦和熔断处理能力）
  • 提高监控预警及时性
  • 恢复策略和方案预演
  • 红蓝对抗，多维度预演生产场景

5.发布上线

除了UAT环境回归，测试同学也要参与需求发布最后环节的工作中去

  • 发布流程把控发布项整理：服务部署、数据库脚本、配置变更、回滚计划等
  • 主动的自动化回归
  • 被动的监控告警配置
  • 争对低峰期的发布：流量回放拨测
  • 最后发布后定期的项目复盘

6.其他

上面说的主要是各个阶段测试的工作职责，除此之外，测试同学应该培养PM职责意识和能力，项目流程管理，持续集成，服务治理等很多方向都可以参与或者加入到工作中去。
最后不要吐槽我的专业程度和全面性，自身也是需要不断学习不断提高，就这样把。
